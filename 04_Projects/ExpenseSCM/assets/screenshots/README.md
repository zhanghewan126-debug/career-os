## 1. 命名规范

按业务流程顺序编号，英文 snake_case，统一 PNG 格式：

```text
assets/screenshots/
├── 01_pr_list.png            请购单列表
├── 02_pr_detail.png          请购单详情
├── 03_ppr.png                采购准备单
├── 04_po.png                 采购订单
├── 05_receiving.png          验收单
├── 06_receiving_exception.png 收料异常单
├── 07_inventory_onhand.png   库存现有量查询
├── 08_issue_request.png      领用申请单
├── 09_return_rtv.png         验退单
├── 10_cost_report.png        月度成本核算报表
└── 11_workflow_approval.png  审批流待办 / 审批节点
```

编号不要求全部凑齐，按实际能截到、且脱敏后合规的画面为准，宁缺毋滥。

## 2. 脱敏检查清单

上传前逐张核对，任何一项未处理都不要提交：

- [ ] 公司名称 / Logo / 内部系统域名（浏览器标签页标题、页面页眉页脚、系统登录页）
- [ ] 真实供应商名称、客户名称
- [ ] 真实员工姓名、工号、部门全称（尤其是审批节点上显示的具体审批人）
- [ ] 真实金额、单价、成本数据（采购单价、成本单位成本等商业敏感信息，建议替换为示例数字）
- [ ] 内部 IP、服务器地址、数据库连接串（尤其容易出现在报错页面截图里）
- [ ] 浏览器书签栏、其他打开的标签页标题
- [ ] 任务栏 / 系统托盘中暴露的其他内部应用图标或通知
- [ ] 单据编号如果是连续可预测的真实业务单号，建议打码或替换为示例编号（如 PR2026000123 → PR-DEMO-001）
- [ ] 截图时间戳如果能反推出具体业务活动，视情况裁剪或打码

推荐用画图工具的马赛克/涂黑而不是简单裁切，避免只是缩小了敏感区域但仍可辨认。
