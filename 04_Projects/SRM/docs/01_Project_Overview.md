# Project Overview｜SRM

---

## 1. Project Introduction

SRM（Supplier Relationship Management，供应商关系管理）是公司 EIP/EIMP（企业信息门户）系统中的核心子系统。

系统覆盖供应商从建立申请到最终绩效评估的完整生命周期，包括：

- 供应商建立申请
- 供应商资料完善
- 多级工作流签核
- 抛转 ERP（AP/PO）
- 供应商资料变更
- 供应商查询与合约附档管理
- 供应商评鉴表与稽核计划
- 供应商配合性评分
- 供应商绩效评分统计
- 体系文件 / 合约 / 代理资格到期提醒

系统采用前后端分离的轻量 Web 架构，后端基于自研 Eip 框架（Python），通过 DB Link 与 Oracle EBS 正式库实时交互，业务逻辑下沉到 Oracle PL/SQL 存储过程包，并接入公司统一的工作流签核引擎与通知渠道，同时支持多组织（境内外子公司）差异化业务规则与中英文双语单据。

---

## 2. Business Background

企业供应商数量多、分布在多家境内外子公司，供应商主数据（基本信息、地址、SITE、联络人、付款资料、体系文件、合约）原本容易出现建档分散、EIP 与 ERP 数据不一致、重复建档、资质文件遗漏等问题。

项目实施前主要存在以下问题：

- 供应商建档缺乏统一查重机制，容易出现重复建档
- 供应商资质文件（体系文件、合约、代理资格）到期缺乏自动提醒，存在合规与履约风险
- EIP 系统录入的供应商数据与 Oracle EBS（ERP）正式库之间缺乏统一的数据一致性机制
- 供应商绩效评估依赖人工统计，跨组织、跨维度的差异化评分规则难以维护
- 不同子公司审批流程、供应商类型规则不同，人工审核成本高

因此，公司决定建设 SRM 模块，实现供应商全生命周期管理的线上化、流程化、自动化。

---

## 3. Project Objectives

项目目标包括：

- 建立统一的供应商全生命周期管理平台
- 打通 EIP 与 Oracle EBS（AP/PO）双库，保障主数据一致性
- 建立供应商查重与强校验机制，从源头保障主数据质量
- 建立多组织差异化的供应商绩效评分体系
- 实现工作流签核驱动的业务自动化（自动建评鉴、自动抛转 ERP、自动通知）
- 支持多组织、多语言（CN/TW 与海外站点）业务规则

## 6. Core Business Value

SRM 将供应商建档、变更、评鉴、抛转 ERP 等原本分散的线下/人工流程统一线上化、流程化、自动化，覆盖多家境内外子公司。

通过查重、强校验与自动回写 ERP，保障了供应商主数据质量与 EIP-ERP 一致性，减少了重复建档与人工录入错误；多维度加权评分统计为采购、品保提供了供应商绩效量化依据与分级管理支撑；体系文件、合约、代理资格到期自动提醒，降低了合规与履约风险。
