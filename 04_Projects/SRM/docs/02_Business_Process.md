# Business Process｜SRM

---

## 1. End-to-End Business Flow

SRM 覆盖供应商从建立申请到绩效评估的完整生命周期。

整体业务流程如下：

```text
采购/需求部门
     │
     ▼
供应商建立申请单（查重 + 动态下拉 + 评鉴前置 + 提交强校验）
     │
     ▼
资料完善（通知供应商联络人补充地址/SITE/联络人/付款资料）
     │
     ▼
多级工作流签核（采购 → 品保/SQE → ⋯）
     │
     ├──────────────────────────────┐
     │                              │
     ▼                              ▼
自动创建评鉴单（建档/新供应商评鉴）   自动生成供应商编码
     │                              │
     └──────────────┬───────────────┘
                     │
                     ▼
          抛转 / 回写 Oracle ERP（AP/PO）
                     │
                     ▼
              供应商建档完成（ONHAND 主数据）
                     │
        ┌────────────┼────────────────────┐
        │            │                    │
        ▼            ▼                    ▼
   资料变更申请   评鉴稽核计划执行      配合性评分（品保）
   （签核后回写）  （评分 + 明细钻取）
        │            │                    │
        └────────────┴──────────┬─────────┘
                                 ▼
                    绩效评分统计（月度/季度，
                    交期/品质/价格/服务四维加权）
                                 │
                                 ▼
                    分级（A/B/C/D）+ 明细钻取
                                 │
                                 ▼
          体系文件 / 合约 / 代理资格到期自动提醒
```

---

## 2. Business Modules

系统由多个相互协作的业务模块组成。

| 模块 | 作用 |
|------|------|
| 建档申请 | 供应商从 0 到 1 建档，含查重、动态下拉、评鉴前置、提交强校验 |
| 资料完善 | 邮件通知供应商联络人补充地址/SITE/联络人/付款资料 |
| 签核流程 | 多级工作流签核（采购、品保、SQE 等） |
| 抛转 ERP | 建档/变更签核通过后自动回写 Oracle EBS（AP/PO） |
| 资料变更 | 已存在于 ERP 的供应商信息新增/修改/失效变更 |
| 查询与合约附档 | 供应商主数据、合约附档、体系文件有效期统一查询 |
| 评鉴管理 | 评鉴单据、稽核计划，支持 Excel 批量导入 |
| 配合性评分 | 品保对供应商服务配合度打分 |
| 绩效评分统计 | 交期/品质/价格/服务四维加权评分与分级 |
| 到期提醒 | 体系文件、合约、代理资格到期定时提醒 |

## 3. Business Stage Description

### ① 供应商建立申请

供应商从 0 到 1 建档的主流程，单据含表头 + 5 个明细 Tab（地址 / 供应商 SITE / 联络人 / 付款资料 / 合约附档）。

系统完成：

- 多源动态下拉：统一封装支持 20+ 种下拉类型（供应商类型、组织、国别、付款/交易条件、币别、税率、银行/支行联动等）
- 供应商查重：简体/繁体/英文名称模糊匹配及统一社会信用代码双重查重，集成启信宝 API 自动拉取并校验企业工商注册信息
- 评鉴前置：建档时自动创建评鉴体系文件（ISO 9001/13485/16949 等）、评鉴选项与评鉴人员
- 提交强校验：依据供应商类型 + 组织动态校验是否维护必要明细、是否上传必传附档

### ② 资料完善

工作流节点触发，向供应商联络人发送资料完善邮件，补充完整地址、SITE、联络人、付款资料等信息。

### ③ 多级工作流签核

按供应商类型、组织路由到采购、品保、SQE 等审批节点；审批通过后，工作流节点钩子驱动 PL/SQL 存储过程自动创建评鉴单、生成供应商编码。

### ④ 抛转 / 回写 ERP

建档、变更经签核后，通过 erp_ / eip_ 双轨标识与中间表机制，将供应商主数据、付款资料、体系文件、联络人抛转/回写 Oracle EBS（AP/PO）。

### ⑤ 资料变更

针对已存在于 ERP 的供应商进行表头/明细（SITE、联络人、付款资料、地址）的新增/修改/失效变更，变更前检查同一对象是否已有在途签核单据，避免并发冲突；跨组织共用的供应商变更会自动提醒其他在用组织的财务。

### ⑥ 评鉴管理

评鉴单据支持按评鉴类别、性质、签核状态查询，签核状态通过递归解析当前多签核人串；稽核计划支持 Excel 批量导入。

### ⑦ 绩效评分统计

对供应商交期、品质、价格、服务四大维度进行综合加权评分与排名统计：

- 多组织差异化权重模型：不同子公司配置不同维度权重与评级阈值（A/B/C/D）
- 动态 SQL 引擎：根据统计维度、显示项、统计周期动态拼接评分项 SQL
- 跨库多源整合：JOIN ERP 收料评分、MES 品检数据、服务评分等多个来源
- 明细钻取：从汇总评分下钻到逐笔收料、检验明细

### ⑧ 到期提醒

系统定时检查体系文件、合约、代理资格等到期情况，自动发送提醒通知，降低合规与履约风险。

## 4. Why This Process Matters

SRM 并不是多个独立模块的简单组合。

整个系统通过：

- erp_ / eip_ 双轨标识 + 中间表 + Merge 回写机制（EIP-ERP 数据一致性）
- 工作流节点钩子驱动的自动化（自动建评鉴、自动生成编码、自动抛转 ERP、自动通知）
- 配置化 + 动态 SQL（多组织差异化绩效评分）
- 组织 × 供应商类型 × 特殊产品维度的强校验规则

将供应商建档、审批、评鉴、绩效评估连接为统一的数据闭环。

每一个业务节点都会驱动主数据状态、签核状态以及 ERP 回写同步更新，保证业务流、数据流和审批流保持一致。因此，该系统不仅解决了供应商管理流程线上化的问题，也建立了企业供应商主数据治理的统一基础。
